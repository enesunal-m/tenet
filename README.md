# tenet

[![CI](https://github.com/enesunal-m/tenet/actions/workflows/ci.yml/badge.svg)](https://github.com/enesunal-m/tenet/actions/workflows/ci.yml)
[![Security](https://github.com/enesunal-m/tenet/actions/workflows/security.yml/badge.svg)](https://github.com/enesunal-m/tenet/actions/workflows/security.yml)
[![License](https://img.shields.io/badge/license-Apache--2.0-blue.svg)](LICENSE)
[![Rust](https://img.shields.io/badge/rust-1.85%2B-orange.svg)](Cargo.toml)

`tenet` compiles structured `.context/` rules into nested `AGENTS.md` files for AI coding agents.

It lets teams keep durable project context as small, typed, reviewable markdown files instead of maintaining one large hand-written agent prompt. Generated `AGENTS.md` files stay compatible with tools that already read them.

## At a Glance

| Area | Details |
| --- | --- |
| Input | `.context/<type>/<slug>.md` rule files |
| Output | Root and nested `AGENTS.md` files |
| Runtime | Local CLI, no network access required |
| Safety | Refuses to overwrite non-empty handwritten `AGENTS.md` files |
| Review model | Rules are normal markdown files reviewed in pull requests |
| CI model | Format, clippy, tests, docs, MSRV, audit, deny, Scorecard |

## Why

Agent context tends to drift when it lives in one manually edited file. `tenet` makes that context:

- scoped to the directories where it applies
- split into stable rule files
- linted for stale, malformed, empty, unsafe, or misplaced rules
- regenerated deterministically
- protected from accidental hand-edits through marker checks and a pre-commit hook

## Status

`tenet` is early v0 software. The CLI is usable and tested, but the file format and command surface may still evolve before 1.0.

Runtime behavior is fully local. The binary does not need network access.

## Install

From this repository:

```bash
cargo install --path .
```

Run without installing:

```bash
cargo run -- <command>
```

Requirements:

- Rust 1.85 or newer
- Git

## Quickstart

Initialize a git repository:

```bash
tenet init
```

This creates `.context/`, `.tenetrc`, an example rule, a generated root `AGENTS.md`, and a pre-commit hook.

Add a rule:

```bash
tenet add invariants \
  --scope "**" \
  --owner enes \
  --priority normal \
  --title "Keep generated files in sync"
```

Write the rule body in your editor:

```markdown
Keep generated files in sync with their source inputs and generation steps.

If a change affects generated output, regenerate and commit the updated
generated files together with the source change.
```

Regenerate agent context:

```bash
tenet compile
```

Check the repo before committing:

```bash
tenet lint --check-compiled
tenet stale
```

Commit both the source rules and generated files:

```bash
git add .context AGENTS.md
git commit -m "Add generated file sync invariant"
```

## Rule Files

Rules live under `.context/<type>/<slug>.md`.

Supported types:

| Type | Use for |
| --- | --- |
| `invariants` | Facts that must not change without agreement |
| `conventions` | Team defaults and preferred patterns |
| `decisions` | Dated decisions with reasoning |
| `gotchas` | Known traps and failure modes |
| `glossary` | Project terms and acronyms |

Example:

```markdown
---
scope: "apps/bundle/**"
owner: platform
reviewed: 2026-04-20
priority: high
tags:
  - auth
---
Session cookies are issued only by the backend.

Do not create client-side session cookies.
```

All frontmatter fields are optional. Missing frontmatter uses defaults.

## Generated Output

Given this rule:

```markdown
---
scope: "**"
owner: platform
priority: normal
---
Run the full test suite before changing public CLI behavior.
```

`tenet compile` writes a generated `AGENTS.md` section like:

```markdown
<!-- generated by tenet from .context/ — do not edit directly -->
<!-- last compiled: 2026-04-21T00:00:00Z -->
<!-- rules: 1 -->

# Project context

## Invariants

- **Run the full test suite before changing public CLI behavior.**

  *owner: platform*
```

Generated files are committed so existing agent tools can read them without knowing about `tenet`.

## Scopes

`scope` is a repo-relative glob. It decides which generated `AGENTS.md` file receives the rule.

| Scope | Result |
| --- | --- |
| `**` | root `AGENTS.md` |
| `apps/bundle/**` | `apps/bundle/AGENTS.md` |
| `apps/*/src/**` | anchored at `apps/` |
| `/apps/**` | invalid because scopes are repo-relative |

## Daily Workflow

Typical contributor loop:

```bash
tenet add conventions --scope "src/**" --owner alice --priority normal --title "Prefer explicit errors"
tenet compile --dry-run
tenet compile
tenet lint --check-compiled
git add .context AGENTS.md
git commit -m "Add explicit error handling convention"
```

Useful review commands:

```bash
tenet list
tenet list --json
tenet list --scope src/lib.rs
tenet show invariants/keep-generated-files-in-sync
tenet review invariants/keep-generated-files-in-sync
tenet stale --json
```

For this repository itself, the project dogfoods `tenet`:

```bash
tenet compile
tenet lint --check-compiled
git add .context AGENTS.md
```

If you change generation behavior, update the tests and regenerate `AGENTS.md` in the same pull request.

## Commands

```bash
tenet init [--force] [--no-hook]
tenet add <type> [--scope GLOB] [--owner NAME] [--priority high|normal|low] [--title TITLE]
tenet list [--type TYPE] [--scope PATH] [--owner NAME] [--stale] [--json]
tenet show <rule-id>
tenet edit <rule-id>
tenet review <rule-id>
tenet stale [--grace DAYS] [--owner NAME] [--json]
tenet compile [--dry-run] [--exclude-stale]
tenet lint [--check-compiled] [--check-secrets] [--quiet]
tenet migrate --from PATH [--yes --mapping FILE]
tenet version
```

In non-interactive use, `tenet add` requires `--scope`, `--owner`, `--priority`, and `--title`.

## Configuration

`.tenetrc` is TOML. Missing config uses built-in defaults.

```toml
[defaults]
grace_days = 90
priority = "normal"

[compile]
header = "<!-- generated by tenet from .context/ — do not edit directly -->"
include_stale = true
segregate_stale = true

[lint]
check_secrets = true
check_filenames = true

[hook]
install_pre_commit = true
```

## Safety

`tenet compile` only overwrites `AGENTS.md` files that are clearly generated by tenet or empty placeholders. Non-empty hand-written `AGENTS.md` files are treated as conflicts.

The pre-commit hook blocks commits when staged `.context/` changes are out of sync with generated `AGENTS.md` files.

## What tenet Does Not Do

`tenet` does not run an MCP server, provide embeddings, sync cross-repo context, or replace agent-specific configuration. Its v0 job is intentionally narrow: keep structured local rule files and generated `AGENTS.md` output in sync.
