# tenet

`tenet` turns structured `.context/` rule files into nested `AGENTS.md` files for AI coding agents.

Instead of hand-maintaining one drifting prompt file, teams keep small reviewed markdown rules under `.context/<type>/`. `tenet compile` renders the `AGENTS.md` tree that existing tools already know how to read.

## Status

`tenet` is early v0 software. The CLI is usable, tested, and intended to be fully offline at runtime.

## Install

From this repository:

```bash
cargo install --path .
```

From a checkout without installing:

```bash
cargo run -- <command>
```

Rust 1.85 or newer is required.

## Quickstart

Initialize a git repository:

```bash
tenet init
```

This creates:

- `.context/` with the five supported rule type directories
- `.tenetrc` with default config
- `.context/invariants/example.md`
- root `AGENTS.md`
- `.git/hooks/pre-commit`, unless `--no-hook` is used

Add a rule:

```bash
tenet add invariants \
  --scope "**" \
  --owner enes \
  --priority normal \
  --title "Run tests before changing behavior"
```

Compile generated agent files:

```bash
tenet compile
```

Check repository health:

```bash
tenet lint --check-compiled
tenet stale
```

Preview compile output without writing:

```bash
tenet compile --dry-run
```

## Rule Files

Rules live at `.context/<type>/<slug>.md`.

Supported types:

- `invariants`
- `conventions`
- `decisions`
- `gotchas`
- `glossary`

Example:

```markdown
---
scope: "apps/bundle/**"
owner: platform
reviewed: 2026-04-20
priority: high
tags:
  - auth
---
Session cookies are issued only by the backend.

Do not create client-side session cookies.
```

All frontmatter fields are optional. A file with no frontmatter uses defaults.

## Scopes

`scope` is a repo-relative glob. It decides where a rule is rendered.

- `**` applies at the repo root.
- `apps/bundle/**` renders in `apps/bundle/AGENTS.md`.
- `apps/*/src/**` anchors at `apps/`.
- Absolute scopes like `/apps/**` are invalid.

## Commands

```bash
tenet init [--force] [--no-hook]
tenet add <type> [--scope GLOB] [--owner NAME] [--priority high|normal|low] [--title TITLE]
tenet list [--type TYPE] [--scope PATH] [--owner NAME] [--stale] [--json]
tenet show <rule-id>
tenet edit <rule-id>
tenet review <rule-id>
tenet stale [--grace DAYS] [--owner NAME] [--json]
tenet compile [--dry-run] [--exclude-stale]
tenet lint [--check-compiled] [--check-secrets] [--quiet]
tenet migrate --from PATH [--yes --mapping FILE]
tenet version
```

In non-interactive use, `tenet add` requires `--scope`, `--owner`, `--priority`, and `--title`.

## Configuration

`.tenetrc` is TOML. Missing config uses these defaults:

```toml
[defaults]
grace_days = 90
priority = "normal"

[compile]
header = "<!-- generated by tenet from .context/ — do not edit directly -->"
include_stale = true
segregate_stale = true

[lint]
check_secrets = true
check_filenames = true

[hook]
install_pre_commit = true
```
